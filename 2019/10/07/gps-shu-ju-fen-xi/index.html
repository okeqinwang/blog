1.清洗数据

getAllUserData(String inpath, String opath)

2.初始化用户数据

选择上面处理过的文件，初始化LetZinkModel 静态成员
List<GpsModel> list
调用初始化方法
setUserGpsData(String inpath)

3.转换GPS坐标为百度坐标系

gps2baidu()

调用此方法需要进行百度开放平台的认证。
配置开发者账号。这里采用配置的形式，在config.propertys文件里面进行设置。

#file config.propertys
url:https://api.map.baidu.com/geoconv/v1/?coords=
from:1
to:5
ak:replace with your ak

4.分割路径

参数：
diserr：距离阈值。路径的起始点距离小于该值，则该路径舍弃。
timeerr：时间阈值。超过该值，开始一条新的路径。
minrange：区域阈值。超过该值，定义一个新的区域标识号。
separateGpsPath(Double diserr,Double timeerr,Double minrange )

建模

可视化

借助百度地图LBS服务，将用户的GPS轨迹数据，显示在地图上，并且绘制出轨迹。
可视化分为两个部分：

  • 包含全部路径的数据表
  • 包含原始数据，不同拆分路径

GeoLife Project Data Process

Geolife是微软开展的一个项目，采集了182位用户从07-12年大约四年内的个人生活GPS轨迹数据，用于学术研究。关于详细的项目信息可以访问微软GeoLife项目。

预处理Geolife数据

原始的文件分散在不同的文件夹里面，而且头部包含一些非结构化的说明，这里采用shell脚本清除掉相关冗余杂乱的数据，最终形成以用户id为文件名的182个文件。里面的数据含义具体表示为：

用户id，纬度，经度，高度，时间戳
000,39.984683,116.31845,492,2008-10-23 02:53:10
000,39.984686,116.318417,492,2008-10-23 02:53:15
000,39.984688,116.318385,492,2008-10-23 02:53:20

处理脚本，需要具有执行权限。

#!/bin/bash

echo 'hell world,begin to deal with this data'

for filename  in `ls /home/spark/geolife/data`
do
    echo $filename
    touch ~/sample/$filename
    find ~/geolife/data/$filename  -name "*.plt" | sort | xargs \
        awk -F ',' '{print(fname","$1","$2","$4","$6" "$7)}'  fname="$filename" >> ~/sample/$filename

done

数据采样

原始的数据采样的频率为5s一次，数据过于密集，增大数据存储开销，也影响数据可视化的性能。更重要的是，这么多的数据在坐标转换为百度坐标系里面的坐标也会带来一系列问题，比如每秒请求次数的限制等等。在geolife整理后数据的基础上，我们进行了抽样处理，由原来的5s一次采样到1分钟一次。具体实现参见GpsDemo项目中edu.cx.util.ResampleGPS.

坐标转换

由原来的GPS坐标体系转换为百度坐标系坐标。采用百度提供的API实现。可以参考百度坐标转换说明文档。

具体实现也可以参考edu.dcy.gps.LetZinkModel里面的 getRightPoints方法。

切分路径
上面baidugps表保存的是用户的所有数据，但是没有切分为具体的路径，为了有较好的扩展性，我们使用一张新的表来实现切分路径，该表里面保存有每个用户每条路径的路径号以及起始时间，我们可以根据不同的参数生成不同的路径划分表，与原始数据表baidugps进行关联查询，于是可以根据用户id以及路径号查询这段时间的轨迹数据进行展示。
具体实现参考edu.cx.LetZinkModel里面的separateGpsPath方法。
下表表示根据距离阈值为500、时间阈值为10生成的路径划分表。

建立路径划分表

create table subpath500_10(
uid int,
pathid int,
begin datetime,
end  datetime,
points int,
dis double
)

字段含义分别为：用户id、路径号、开始时间、结束时间、该路径包含的gps记录点数目、起始点距离。便于展示，其中包含gps记录点数以及起始点距离没有列出。
